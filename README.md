# crosstalk

LLM駆動の恋愛ドロドロ群像劇シミュレーションゲーム。サーバ不要、ブラウザ完結。

舞台は、テニスサークルという**閉じた人間関係のコミュニティ**。キャラクター全員(成人)に、交際・浮気・元恋人・片想い・誰にも言えない秘密といった火種が最初から仕込まれています。プレイヤーは好きなキャラに乗り移り、一日ごとに行動計画を決めてターンを進めると、LLMがゲームマスターとしてその日を執行します。

- キャラ全員が自分の秘密と目標を持ち、勝手に動き、勝手に拗れる。秘密は少しずつ漏れ、疑念→綻び→言い逃れ→決定的瞬間と段階を踏んで修羅場が爆発する
- 表のグループチャットでは平和を装い、DMで裏が動く。自分が参加していないDMも**覗き見**できる
- 全員が**鍵垢のSNS**を持ち、誰にも見せるつもりのない本音・愚痴・恨みつらみを吐き出している——それも神の視点で覗ける
- 「**神の采配**」で今日必ず起こる出来事を強制したり、失敗した日は朝に**巻き戻して**やり直したりできる
- 最後は「**終幕**」で全DM・全員の秘密・鍵垢の本音まで全記録が晒される、全暴露の総括が生成される

舞台とキャスト(誰が誰と何を隠しているか)は**シナリオパック**としてエンジンから分離されており、「新しいゲーム」でシナリオを選んで遊べます。YAMLを2枚書くだけで自作の舞台を追加できます。

## 遊び方の流れ

1. 「新しいゲーム」で**シナリオを選ぶ**。**メンバー** タブでキャラと関係性を眺め、好きなキャラに**乗り移る**
2. **行動計画** タブで今日の計画を書く(自分で書く / AIに考えてもらう / AI案を編集する)。必ず起こしたいことがあれば「神の采配」へ
3. 「一日を進める」を押すと、NPC全員の計画が裏で生成され、一日がシミュレートされる
4. **できごと** タブで特筆すべき場面の短編小説と関係の変化を読む。やり直したい日は朝に巻き戻せる
5. **メッセージ** タブでグループチャット・DMを覗き見る。自分が参加しているトークには発言して介入もできる(参加していないDMは覗き見のみ)
6. **鍵垢** タブで各キャラの誰にも見せない本音を覗く。自分の鍵垢に書いた本音はゲームマスターが汲み取って翌日に反映する
7. 物語を畳みたくなったら **終幕** タブで全暴露の総括を生成する

## セットアップ・起動

```sh
npm install
npm run dev      # http://localhost:5173
```

起動したら画面右上の「⚙設定」から **Anthropic APIキー** を入力します(現状 Anthropic 専用)。
キーはこのブラウザの localStorage にのみ保存され、Anthropic API 以外には送信されません。
モデルも同じ設定画面で切り替えられます(既定: claude-opus-4-6、ストーリーテリング力の観点からOpus4.6を推奨します)。

## テスト・ビルド・配布

```sh
npm run smoke    # 純ロジック(状態要約・シミュレーション適用・名前解決)のテスト
npm run build    # dist/ に静的サイトを出力
npm run preview  # ビルド結果の確認
```

ビルド成果物は完全な静的サイトなので、GitHub Pages 等にそのまま置けます。
遊ぶ人は各自のAPIキーを⚙設定に入れる形になります。

## セーブデータ

セーブは**ブラウザ側**に自動保存されます(本体は localStorage、巻き戻し用の日次スナップショットは IndexedDB)。
ブラウザ(プロファイル)ごとに独立したセーブになります。
ヘッダーの「💾 書き出し / 📥 読み込み」でセーブのバックアップ・復元・引っ越しができます。
ブラウザのサイトデータを消すとセーブも消えるので、大事なプレイは書き出しておくこと。

## シナリオパック

コード(汎用エンジン)とシナリオ(世界観・キャスト)は分離されています。
新しいシナリオは `scenarios/<id>/` にYAMLを2枚置くだけで作れます(コメントも書けます)。
ビルド時に同梱されるので、追加後は dev サーバの再起動(またはリビルド)が必要です。

```
scenarios/loveall/
  scenario.yaml       — 世界観プロンプト・トーン・生活リズムのルール・
                        終幕の演出・UI表示(ブランド名/計画のプレースホルダ)
  initial_state.yaml  — キャスト・初期関係(間柄/片想い/肉体関係回数)・
                        スレッド(kind: group が全体チャット)・初期メッセージ・初期投稿
```

`scenario.yaml` の `world_prompt_by_model` で、使用モデルに応じた世界観プロンプトの出し分けができます
(キーはモデル名に対するglobパターン。マッチしなければ `world_prompt` を使用)。

セーブデータは自分がどのシナリオ生まれかを覚えているので、
別シナリオのセーブを読み込んでもプロンプトは正しい世界観で動きます。

## デバッグ

LLMログタブで、LLMに送った生のプロンプトと応答(直近50件・IndexedDB保存)を確認できます。
用途ラベル・所要時間・トークン数付き。

## 構成

```
index.html       — UI
src/
  main.js        — UI制御(タブ・モーダル・進捗・既読管理)
  engine.js      — ゲームエンジン(状態要約・計画生成・一日のシミュレーション・
                   チャット返信・イベント管理・終幕)
  llm.js         — Anthropic SDK ラッパー(JSON抽出・リトライ・ストリーミング進捗・
                   デバッグログ)
  scenario.js    — シナリオパックの読み込み(ビルド時に同梱)
  state.js       — 状態ヘルパ・localStorage/IndexedDB 永続化
  style.css      — スタイル
scenarios/       — シナリオパック(世界観+初期キャスト)
smoke.mjs        — node で動くスモークテスト
```
