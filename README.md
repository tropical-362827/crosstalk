# crosstalk

LLM駆動の恋愛ドロドロ群像劇シミュレーションゲーム。

舞台は、**閉じた人間関係のコミュニティ**。プレイヤーは好きなキャラに乗り移り、一日ごとに行動計画を決めてターンを進めると、LLMがゲームマスターとしてその日を執行します。

- キャラ全員が自分の秘密と目標を持ち、勝手に動き、勝手に拗れる。秘密は少しずつ漏れ、疑念→綻び→言い逃れ→決定的瞬間と段階を踏んで修羅場が爆発する
- 表のグループチャットでは平和を装い、DMで裏が動く。自分が参加していないDMも**覗き見**できる
- 全員が**鍵垢のSNS**を持ち、誰にも見せるつもりのない本音・愚痴・恨みつらみを吐き出している——それも神の視点で覗ける
- 「**神の采配**」で今日必ず起こる出来事を強制したり、失敗した日は朝に**巻き戻して**やり直したりできる
- 最後は「**終幕**」で全DM・全員の秘密・鍵垢の本音まで全記録が晒される、全暴露の総括が生成される

舞台とキャスト(誰が誰と何を隠しているか)は**シナリオパック**としてエンジンから分離されており、「新しいゲーム」でシナリオを選んで遊べます。YAMLを2枚書くだけで自作の舞台を追加できます。

## 遊び方の流れ

1. 「新しいゲーム」で**シナリオを選ぶ**。**メンバー** タブでキャラと関係性を眺め、好きなキャラに**乗り移る**
2. **行動計画** タブで今日の計画を書く(自分で書く / AIに考えてもらう / AI案を編集する)。必ず起こしたいことがあれば「神の采配」へ
3. 「一日を進める」を押すと、NPC全員の計画が裏で生成され、一日がシミュレートされる
4. **できごと** タブで特筆すべき場面の短編小説と関係の変化を読む。やり直したい日は朝に巻き戻せる
5. **メッセージ** タブでグループチャット・DMを覗き見る。自分が参加しているトークには発言して介入もできる(参加していないDMは覗き見のみ)
6. **鍵垢** タブで各キャラの誰にも見せない本音を覗く。自分の鍵垢に書いた本音はゲームマスターが汲み取って翌日に反映する
7. 物語を畳みたくなったら **終幕** タブで全暴露の総括を生成する

## セットアップ

[uv](https://docs.astral.sh/uv/) を使います。

```sh
uv sync
```

`.env.sample` をコピーして `.env` を作り、LLMのAPIキーを設定します(使うプロバイダに応じて):

```sh
cp .env.sample .env
```

```dotenv
ANTHROPIC_API_KEY=sk-ant-...  # Anthropic の場合
# OPENAI_API_KEY=sk-...       # OpenAI の場合
# GEMINI_API_KEY=...          # Gemini の場合
```

## 起動

```sh
uv run uvicorn server.app:app --port 8000
```

ブラウザで http://localhost:8000 を開く。

## LLMの切り替え

`config.yaml` の `model` を [litellm のモデル指定](https://docs.litellm.ai/docs/providers) に書き換えるだけです(サーバー再起動が必要)。

```yaml
model: anthropic/claude-sonnet-4-6  # 例: openai/gpt-4o-mini, gemini/gemini-2.0-flash, ollama/llama3
temperature: 0.9
max_tokens: 8000
```

ただ、Claudeモデル以外だと情緒的な描写がいまいちな感があります。(主観的意見)

設定(`config.yaml`、コミット可)と秘密情報(`.env`、gitignore済み)は分離しています。APIキーは必ず `.env` 側へ。

## セーブデータ

セーブは**ブラウザ側**に自動保存されます(本体は localStorage、巻き戻し用の日次スナップショットは IndexedDB)。
サーバはステートレスで何も保存しないため、ブラウザ(プロファイル)ごとに独立したセーブになります。
画面右上の「新しいゲーム」でシナリオを選んで最初からはじめられます。
ヘッダーの「💾 書き出し / 📥 読み込み」でセーブのバックアップ・復元・引っ越しができます。
ブラウザのサイトデータを消すとセーブも消えるので、大事なプレイは書き出しておくこと。

## シナリオパック

コード(汎用エンジン)とシナリオ(世界観・キャスト)は分離されています。
`config.yaml` の `scenario` で使うシナリオを指定します(変更後はサーバー再起動+新しいゲーム)。

新しいシナリオは `scenarios/<id>/` にYAMLを2枚置くだけで作れます(Pythonは触りません。コメントも書けます):

```
scenarios/loveall/
  scenario.yaml       — 世界観プロンプト・トーン・生活リズムのルール・
                        終幕の演出・UI表示(ブランド名/計画のプレースホルダ)
  initial_state.yaml  — キャスト・初期関係(間柄/片想い/肉体関係回数)・
                        スレッド(kind: group が全体チャット)・初期メッセージ
```

シナリオパックはYAML専用です。セーブデータ(ブラウザ保存・書き出しファイル)はJSONです。

`scenario.yaml` の `world_prompt_by_model` で、使用モデルに応じた世界観プロンプトの出し分けができます
(キーは `config.yaml` の `model` に対するglobパターン。例: `"xai/*"`。マッチしなければ `world_prompt` を使用)。

セーブデータは自分がどのシナリオ生まれかを覚えているので、
別シナリオのセーブを読み込んでもプロンプトは正しい世界観で動きます。

## 構成

```
server/
  app.py      — FastAPI(ステートレスAPI + 静的配信 + バックグラウンドジョブ)
  engine.py   — 汎用ゲームエンジン(状態要約・計画生成・一日のシミュレーション・
                チャット返信・イベント管理・終幕)
  scenario.py — シナリオパックの読み込み
  llm.py      — litellm ラッパー(JSON抽出・リトライ・ストリーミング進捗)
  models.py   — Pydantic モデル
scenarios/    — シナリオパック(世界観+初期キャスト)
web/          — フロントエンド(素のHTML/CSS/JS。セーブの正本は
                localStorage / IndexedDB に保存)
```
